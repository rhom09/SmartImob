# Plano de Implementação: Multi-tenancy (Fase 7.1)

## Contexto
Atualmente, o sistema SmartImob não possui isolamento de dados entre diferentes imobiliárias. Todos os usuários acessam o mesmo conjunto de dados no banco de dados. O objetivo é implementar Multi-tenancy, garantindo que cada imobiliária tenha acesso apenas aos seus próprios imóveis, contratos, clientes e configurações.

## Abordagem Recomendada

1. **Definição do Tenant**: A entidade `Imobiliaria` será o tenant raiz.
2. **Isolamento de Dados**:
    - Adicionar `imobiliariaId` em todos os modelos principais (`Property`, `Tenant`, `Contract`, `Owner`, `User`, `Expense`, `Receipt`).
    - Adicionar `imobiliariaId` à tabela `User` para vincular o funcionário/gestor à imobiliária.
3. **Segurança (Backend)**:
    - Aplicar filtros `where: { imobiliariaId: req.user.imobiliariaId }` em todos os repositórios (dashboardService, imoveisService, etc.).
4. **Segurança (Banco de Dados - RLS)**:
    - Opcional inicialmente (via Prisma), mas recomendado futuro para RLS no Supabase.

## Arquivos Críticos para Modificação

### Banco de Dados
- `backend/prisma/schema.prisma`: Adicionar modelo `Imobiliaria` e campo `imobiliariaId` nos modelos.

### Backend
- `backend/src/middleware/auth.ts`: Atualizar para buscar `imobiliariaId` do `req.user` (extraído do banco após o login).
- Todos os Services (`backend/src/services/*.ts`): Adicionar filtro de `imobiliariaId` em todas as queries.

### Frontend
- `frontend/src/components/Navigation.tsx`: Ajustar exibição de menu se necessário.

## Plano de Verificação (Fase 6.1)

1. **Execução de Migração**: Aplicar `prisma migrate dev` para atualizar o schema.
2. **População de Dados**: Criar a imobiliária padrão e migrar usuários existentes.
3. **Teste Funcional**:
    - Logar com usuário A (Imobiliária A), listar imóveis e verificar se aparecem apenas imóveis da A.
    - Logar com usuário B (Imobiliária B), listar imóveis e verificar o isolamento.
4. **Verificação de Acesso**: Tentar acessar rotas de API diretamente (ex: via Postman) forçando um ID que não pertence à imobiliária e validar bloqueio (403/404).
