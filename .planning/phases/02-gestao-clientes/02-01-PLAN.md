<task type="auto" tdd="true">
  <name>Task 1: Atualizar Schema Prisma para Clientes e Interações</name>
  <files>backend/prisma/schema.prisma</files>
  <behavior>
    - O modelo Tenant deve ter um campo 'tipo' (enum ou string) para diferenciar Inquilino de Interessado.
    - Criar modelo Interaction para registrar histórico (data, tipo, descricao, responsavel).
    - Garantir que Tenant possui campos para CPF/CNPJ, Telefone, Email e Endereço conforme DER.md.
  </behavior>
  <action>
    - Adicionar enum `TipoCliente { INQUILINO, INTERESSADO }` ao schema.
    - Atualizar model `Tenant`: adicionar `tipo TipoCliente @default(INQUILINO)`, `rg String?`, `dataNascimento DateTime?`.
    - Criar model `Interaction`:
      ```prisma
      model Interaction {
        id           String   @id @default(uuid())
        tenantId     String   @map("cliente_id")
        usuarioId    String   @map("usuario_id")
        tipo         String   // Ligação, Email, Visita, etc.
        descricao    String
        data         DateTime @default(now())
        tenant       Tenant   @relation(fields: [tenantId], references: [id])
        usuario      User     @relation(fields: [usuarioId], references: [id])
        @@map("INTERACOES")
      }
      ```
    - Adicionar relação `interacoes Interaction[]` ao model `Tenant`.
    - Executar `npx prisma generate`.
  </action>
  <verify>
    <automated>npx prisma validate</automated>
  </verify>
  <done>Schema atualizado e validado com suporte a tipos de cliente e histórico de interações.</done>
</task>
